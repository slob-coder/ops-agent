# 授权规则

## 默认策略
- 所有只读观察命令（L0）：直接执行，不需要通知
- 写 Notebook（L1）：直接执行
- 重启非核心服务（L2）：通知人类后直接执行
- 重启核心服务（L2 核心）：必须人类批准
- 修改配置文件（L2）：通知人类后执行，必须先备份原文件
- 修改代码/提交 PR（L3）：必须人类批准
- 删除文件、drop 数据库、格式化磁盘等破坏性操作（L4）：永远拒绝

## 核心服务列表
以下服务的重启/停止必须人类批准：
- mysql / mariadb / postgresql（数据库）
- redis / memcached（缓存）
- nginx / haproxy（网关/负载均衡）
- docker / containerd（容器运行时）

## 信任度演化
- 入职前 7 天：所有 L2 操作都需要人类批准
- 连续成功 10 次 L2 操作后：非核心服务 L2 操作降级为"通知后执行"
- 任何一次 L2 操作失败导致回滚：信任度回退，L2 重新需要批准，持续 5 次成功后恢复

## 紧急情况
如果判断系统正在发生严重故障（如服务完全不可用、数据有丢失风险），
可以在通知人类的同时直接执行 L2 操作，不等待批准。
但事后必须在 Incident 中详细记录理由。
